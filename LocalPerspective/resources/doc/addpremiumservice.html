Configure a Premium Service for anonymous access

This configurable app supports the configuration of the premium route service to support anonymous access scenarios like providing driving directions to a fixed destination or locating the closest site to the userâ€™s location and routing them to that location.

Before you can publish and share your app, you need to follow the steps below to configure the route service for anonymous access.

  1. Create a new item in ArcGIS Online or your portal
    • Sign in to your organization and verify that you have privileges to create content.
    • Open My Content and click Add Item.
    • Choose On the Web from the The item is option.
    • Enter this URL to the World Route service:
      http://route.arcgis.com/arcgis/rest/services/World/Route/NAServer
    • Enter the Username and Password for an organizational account that has access to the service and select the option to store the credentials with the service item. If credentials are stored with the item, users will not be prompted for credentials when using the routing functionality in your app.
    • Enter a title and tags for the item.
    • Click Add Item after verifying that the dialog looks similar to this:
  2. Share the item with everyone. Sharing it with just the groups in your organization will still require users to login to access it.
  3. Limit usage of the route service you just added by defining rate limiting and referrer.
    • On the item details page, click Edit then under Advanced Options click Limit Usage.
    • Check Enable rate limiting and specify the number of requests per time period you want to allow.
    • Specify the Referrer URL or IP to further restrict traffic. You can provide either the URL to your app, or if you plan to have multiple configurable apps that use this service you can specify the URL the configurable app root directory instead. The configuratble app root directory will look something like this: http://www.arcgis.com/apps.
  4. Copy the URL for the item from the ArcGIS Web REST Connection section. You will need this URL when configuring the app.
  5. Return to the template configuration panel and configure the app to use the secure service by pasting the URL you copied in step 4 into the Route utility item field.
